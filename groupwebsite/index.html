Project Overview: BakeTrack API

The BakeTrack API will serve as a complete backend solution for a bakeryâ€™s online ordering process, supporting both staff and customer operations through a RESTful, database-driven architecture.

Project Goal

Develop an API that supports all core bakery operations: order management, menu control, ingredient tracking, and customer feedback.

Primary Functions

Staff Perspective

  • Create, update, and delete menu items.
  • Monitor inventory and receive low-stock alerts.
  • View all orders or specific order details.
  • Track dish popularity and review summaries.
  • Manage promotions and expiration dates.
  • View revenue and order history by date range.

Customer Perspective

  • Place orders as guests.
  • Select delivery or pickup options.
  • Pay for orders and track order status.
  • Search menu items by type or dietary preference.
  • Submit reviews and ratings.
  • Apply promotional codes.

Technical Scope

Architecture: BakeTrack follows a RESTful design using SQLAlchemy with a MySQL database. Swagger UI is used for documentation and testing.

Core Tables: menu_items, orders, order_items, customers, reviews, promotions, ingredients, staff, inventory

Planned Features:

  • CRUD for all core tables
  • Inventory checks before confirming orders
  • Data reporting endpoints (revenue, popular items)
  • Unit testing with pytest
  • Preloaded test data for demonstrations
